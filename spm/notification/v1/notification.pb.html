Services

NotificationService

Notification Service

The notification service provides access to notifications.

rpc ListNotifications(ListNotificationRequest) returns (ListNotificationResponse)

List Notifications. Notifications must be filtered by user and optionally by read status.

rpc UpdateNotificationsReadStatus(UpdateNotificationReadStatusRequest) returns (google.protobuf.Empty)

Update notifications read status.

Types

ListNotificationRequest

List notification request.

Field Type Description
tenant string
user string

User name that has notification.

Type of notification.
status Status

Read status of notification.
pageSize int32

Maximum number of results to be returned. Default is 20.
pageToken string

Page token to request a specific page of the list results. Set to “” for first page.

ListNotificationResponse

Field Type Description
notifications Notification[]
nextPageToken string

Pagination token to retrieve the next page of results. If the value is “”, it means no further results for the request.

Notification

Notification

Notification sent to user when something of interest happened. Currently all notifications are sent to user. In the future we might want to have notification settings where user can watch or subscribe to specific resource or event.

Field Type Description
id uint64

Notification id. auto generated.
name string

Notification name. auto generated. Name will be set to tenants/tetrate/notifications/id.
user string

User name that own this notification.
tenant string

Tenant. Tenant is obtained from user.
message string

Notification message.
type NotificationType

Notification type.
read bool

Notification read status.
resourceName string

Resource name is resource name that notification refer to. For example if notification triggered because of new alert, resource_name is alert name.
timestamp google.protobuf.Timestamp

Time when the notification was created.

NotificationType

Notification type could be RESOURCE or ALERT. Alert is one type of resource. We differentiate type here to allow different user experience for “normal” event and Alert.
