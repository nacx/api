Services

AlertService

Alert Service

The alert service provides access to alerts. In TSB, Alert is defined in Application. Access to Alert is allowed for any role that has READ access to Application.

rpc ListAlerts(ListAlertsRequest) returns (ListAlertsResponse)

List alerts. Alert can be filtered by both Status and Priority.

rpc GetAlert(GetAlertRequest) returns (Alert)

Get single alerts.

rpc UpdateAlert(UpdateAlertRequest) returns (google.protobuf.Empty)

Update alert. User can update alert status, ack, and owners.

rpc DeleteAlert(DeleteAlertRequest) returns (google.protobuf.Empty)

Delete alert. This will remove alert from storage.

Types

Alert

Alert

Alert that created because of alert rule violation. Alert is defined as a resource under tenant.

Field Type Description
id uint64

Alert id. auto generated.
name string

Alert name. auto generated. Name will be set to tenants/tetrate/alerts/id.
tenant string

Tenant. Tenant is obtained from application name where alert rule defined.
ruleName string

Rule name that trigger this alert.
application string

Application name where the rule belong to.
message string

Alert message.
status AlertStatus

Alert status.
acknowledged bool

Alert acked or not. When alert acked, it means someone is working on it and notification will not be sent.
timestamp google.protobuf.Timestamp

Time when the alert was triggered.
severity tetrate.api.tcc.spm.alert.v1.Severity

Alert severity.
owners string[]

Alert owner is user responsible to resolve alert. User who acked will be added to owners.
properties map<string, string>

Key value pairs with additional information for the alert.
