Rule

Rule

An alert rule defines the condition when an alert should be triggered.

Field Type Description
id string

Short ID for this rule. Id is generated by client. spmserver will add suffix “_rule” before sending rule to skywalking and remove it when receiving message.
metricName string

Metric name to be watched. Metric name is defined in https://github.com/tetrateio/tetrate/blob/master/install/helm/tcc/charts/oap/templates/01-config.yml#L125-L139.
threshold uint32

The threshold value of a metric; depends on the operator, it triggers alert if the metric value is not match.
operator Operator

The operator. E.g. greater-than, lower-than and equal.
period google.protobuf.Duration

The period of an alert rule should be checked.
count uint32

The number of “violation” in a certain period.
silencePeriod google.protobuf.Duration

Keep silence for some amount of time after an alarm is triggered. By default, the value is the same as period.
displayName string

Friendly name for a rule.
enabled bool

Whether this rule is enabled or disabled. Default to false or disabled.
includeServices string[]

Included services in this rule. This field is deprecated. Use included_services that has validation.
excludeServices string[]

Excluded services from this rule. This field is deprecated. Use excluded_services that has validation.
includedServices Rule.Service[]

Included services in this rule. This will be translated into include-names in skywalking configuration response.
excludedServices Rule.Service[]

Excluded services from this rule. This will be translated into exclude-names in skywalking configuration response. spmserver will omit this from response if empty.
severity Severity

Alert severity. Default to MEDIUM.
thresholds Rule.AlertThreshold[]

Thresholds value as array to set multiple threshold values. If this value is set, single value threshold will be ignored

Rule.AlertThreshold

Field Type Description
value string

The threshold value of a metric; depends on the operator, it triggers alert if the metric value is not match.

Rule.Service

Service to be included or excluded.

Field Type Description
name string

Service name. Service name follows SPM noun. https://github.com/tetrateio/SPM/blob/spm2/doc/spm-nouns.md#mapping Example: subset|service|namespace|cluster|env, |service|namespace||env, subset|service|namespace||env. Allowed characters are 0-9, a-z, .~_-.
