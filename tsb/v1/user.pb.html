User represents a user that has been loaded from a configured Identity Provider (IdP) that can log into the paltform. Currently, users are automatically synchronized by TSB from a configured LDAP server.

The following example creates a user named john under the tenant mycompany .

apiVersion: api.tsb.tetrate.io/v1
kind: User
metadata:
  name: john
  tenant: mycompany
spec:
  username: john
  firstName: John
  lastName: Doe
  displayName: John Doe
  email: john.doe@acme.com

User

User represents a user from the identity Provider that is allowed to log into the platform.

Field Type Description Required
loginName string

The username used in the login credentials.

 Yes
firstName string

The first name of the user.

 No
lastName string

The last name of the user, if any.

 No
displayName string

A human friendly display name for the user.

 No
email string

Email for the user where alerts and other notifications will be sent.

 No
