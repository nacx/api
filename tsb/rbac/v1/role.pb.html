Role is a named collection of permissions that can be assigned to any user or team in the system. The set of actions that can be performed by a user, such as the ability to create, delete, or update configuration will depend on the permissions associated with the user’s role. Roles are tenant level resources that are defined once. RoleBindings in each configuration group will bind a user to a specific role defined apriori.

The following example declares a workspace-admin role with the ability to create, delete configurations and the ability to set RBAC policies on the groups within the workspace.

apiVersion: rbac.tsb.tetrate.io/v1
kind: Role
metadata:
  name: role1
  tenant: mycompany
spec:
  rules:
  - types:
    - apiGroup: api.tsb.tetrate.io/v1
      kinds:
      - workspaceSetting
    permissions:
    - create
    - read
    - delete
    - write
    - set_rbac
  - types:
    - apiGroup: traffic.tsb.tetrate.io/v1
    - apiGroup: security.tsb.tetrate.io/v1
    - apiGroup: gateway.tsb.tetrate.io/v1
    permissions:
    - create
    - delete
    - read
    - write
    - set_rbac

Role

Role is a named collection of permissions that can be assigned to any user or team in the system.

Field Type Description Required
rules Rule[]

A set of rules that define the permissions associated with each API group.

 No

Subject

Subject identifies a user or a team under a tenant. Roles are assigned to subjects for specific resources in the system.

Field Type Description Required
user string (oneof)

The user extracted from LDAP server.

 No
team string (oneof)

A team defined in service bridge.

 No

Role.ResourceType

The type of API resource for which the role is being created.

Field Type Description Required
apiGroup string

A specific API group such as traffic.tsb.tetrate.io/v1.

 Yes
kinds string[]

Specific kinds of APIs under the API group. If omitted, the role will apply to all kinds under the group.

 No

Role.Rule

A rule defines the set of api groups

Field Type Description Required
types ResourceType[]

The set of API groups and the api Kinds within the group on which this rule is applicable.

 Yes
permissions Permission[]

The set of actions allowed for these APIs.

 Yes

Permission

A permission defines an action that can be performed on a resource. By default access to resources is denied unless an explicit permission grants access to perform an operation against it.

Name Description
INVALID

Default value to designate no value was explicitly set for the permission.
READ

The read permission grants read-only access to the resource.
WRITE

The write permission allows the subject to modify an existing resource.
CREATE

The create permission allows subjects to create child resources on the resource.
DELETE

The delete permission grants permissions to delete the resource.
SET_RBAC

The set-rbac permission allows subjects to manage the access policies for the resources.
